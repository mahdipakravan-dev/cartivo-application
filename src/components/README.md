# ماژول `components` — کامپوننت‌های UI

## این بخش دقیقاً چه می‌کند؟

دو دسته کامپوننت سراسری را نگه می‌دارد:

| پوشه | محتوا |
|---|---|
| `ui/` | کامپوننت‌های **shadcn/ui** (فعلاً `button` و `card`). این‌ها با CLI رسمی shadcn قابل گسترش‌اند: `npx shadcn@latest add dialog` و… . کد داخل پروژه کپی می‌شود (نه پکیج)، پس مالک کامل آن هستید. |
| `layout/` | هدر (`site-header.tsx`) و فوتر (`site-footer.tsx`) سراسری — هر دو Server Component بدون JS کلاینتی. |

## قراردادها

- کامپوننت‌هایی که **فقط در یک صفحه** استفاده می‌شوند اینجا قرار نمی‌گیرند؛
  در `_components` کنار همان route می‌مانند (نمونه: `app/brands/_components`).
- توکن‌های رنگ/شعاع همه از CSS Variables در `app/globals.css` می‌آیند —
  رنگ خام (`bg-blue-500`) در کامپوننت‌ها ممنوع است تا تم دارک/برندینگ
  بعداً بدون دست‌زدن به کامپوننت‌ها قابل تغییر باشد.
- RTL: از کلاس‌های logical تیلویند (`ms-*` ،`me-*` ،`start-*`) به‌جای
  `ml/mr` استفاده کنید تا با `dir="rtl"` درست کار کنند.

## وابستگی به ماژول‌های Shared

- `lib/utils.ts` ← تابع `cn` (همه‌ی کامپوننت‌های ui)
- `lib/config/site.ts` ← نام برند در هدر/فوتر
